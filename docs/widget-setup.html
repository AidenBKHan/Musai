위젯 적용 가이드

여러분의 사이트(예: 여행 예약 사이트)에 Musai 안전지수 카드를 붙이는 방법입니다.

← 홈으로

1. 값 입력하고 스니펫 생성

나라 코드는 필수, 지역명은 화면에만 표시되는 선택 값입니다. API 서버 주소를 비워두면 배포 전에도 안전하게 데모 모드로 표시됩니다. 마스코트 이미지는 별도 파일이 아니라 musai-widget.js 안에 내장되어 있어서, 아래 스니펫의 script src만 실제로 접근 가능한 주소로 두면 이미지까지 그대로 따라옵니다.

2. 옵션 설명

속성필수설명
data-country필수목적지 ISO 3166-1 alpha-2 국가 코드 (예: FR, JP, KH)
data-region선택화면에 표시할 지역/도시명 (표시 전용, 비우면 백엔드 값 사용)
data-layout선택card(기본) · bottomsheet(바우처·예약완료 화면용) · wide(데스크톱 사이드바용 — 뚱뚱한 박스가 아니라 마스코트·점수·태그·Safe-How·버튼이 한 줄에 펼쳐지는 길쭉한 형태, 최소 너비 640px) · bubble(화면 구석에 고정된 작은 아이콘, 클릭하면 바텀시트로 펼쳐짐) · banner(예약완료 화면 등 페이지 콘텐츠 안에 놓는 CTA 배너, 클릭하면 바텀시트로 펼쳐짐 — 제안서 목업과 동일한 형태)
data-position선택bubble일 때: 고정할 화면 모서리 — bottom-right(기본) · bottom-left · top-right · top-left. banner일 때: inline(기본, 페이지에 놓은 자리 그대로) · top/bottom(쿠키 동의창처럼 화면 위/아래에 고정)
data-size선택bubble/banner의 크기 — sm · md(기본) · lg. 내부 요소가 비율대로 함께 커지고 작아져서 어떤 크기든 깨지지 않습니다.
data-close-to선택bottomsheet를 바로 지정했을 때만 의미가 있음(버블·배너에서 펼친 경우는 항상 자기 자신으로 접힘) — hide(기본, 닫기를 누르면 완전히 사라짐) · banner/bubble(닫기를 누르면 그 모양으로 접힘)
data-lang선택ko · en — 생략하면 방문자 기기(브라우저)의 언어 설정을 자동 감지해서 적용하고(접속 위치·IP 아님), 한국어가 아니면 영어로 표시됩니다. 위젯 문구와 데모 데이터가 함께 바뀝니다.
data-api-base선택배포된 Musai 백엔드 주소. 비워두면 데모 모드로 표시

요청이 3초 안에 응답하지 않거나 실패해도 자동으로 데모 모드로 전환되어, 백엔드 상태와 무관하게 여러분의 페이지는 깨지지 않습니다.
